# Contributing to Pluginator

Thank you for your interest in contributing to Pluginator!

## How to Contribute

### Reporting Bugs

1. Check [existing issues](https://github.com/NindroidA/pluginator/issues) first
2. Open a [bug report](https://github.com/NindroidA/pluginator/issues/new?template=bug_report.md)
3. Include:
   - Pluginator version
   - Operating system
   - Steps to reproduce
   - Expected vs actual behavior
   - Relevant logs

### Requesting Features

1. Check [existing feature requests](https://github.com/NindroidA/pluginator/issues?q=is%3Aissue+label%3Aenhancement)
2. Open a [feature request](https://github.com/NindroidA/pluginator/issues/new?template=feature_request.md)
3. Describe your use case clearly

### Documentation Improvements

Documentation PRs are welcome! You can submit PRs to improve:
- README.md
- User guide
- Examples

### Code Contributions

Pluginator is closed-source software. Code contributions are not accepted at this time.
If you have ideas for improvements, please open a feature request issue.

## Code of Conduct

Be respectful and constructive in all interactions.
